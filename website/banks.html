Banks and Sets

NOME Definition

A bank statement is used to set values that can be dynamically set through sliders once the NOME file has been compiled. Each bank will be a separate window in Nome with a list of sliders. A slider in a bank can be created by creating a set then specifying the name, value, start, end, and step size of the slider. The value of a bank can then be passed in to a generator call with the syntax {expr $id_setId} where id is the id of the bank and setId is the id of the variable to be assigned.

bank id set setId1 value1 start1 end1 stepSize1 ... set setIdN valueN startN endN stepSizeN endbank

Each setId must be a string representing the name of the variable to be changed. value represents the initial value of the slider. start represents the minimum value of the slider. end represents the maximum value of the slider. stepSize represents by how much the slider must increment.

Example

bank lefttunP set n 8 3 10 1 set ro 0.3 0.1 1.0 0.1 set ratio 0.2 -0.5 0.5 0.1 set h 0.1 0.1 1.5 0.1 endbank

Defines a bank named lefttunP with variable n, ro, ratio, and h. The first variable in the bank can be assigned to a generator using {expr $lefttunP_n}. For example, a point can be created using:

point p1 (0 0 {expr $lefttunP_n}) endpoint

where the variable n would be assigned to the slider controlling the z-position of p1.
