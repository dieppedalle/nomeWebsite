NOME Language Specification
Basic Geometry

Surface

NOME Definition

A surface statement is used to define the material of an object which can then be referenced by its id.

surface id color ( color_triple ) endsurface

The point_triple defines the RGB color of the surface. The values of each of the three coordinate must be separated by spaces and between 0 and 1.

Example

surface sBlue color (0 0 1) endsurface

Defines a surface named sBlue with a blue color.

Point / Control Point

NOME Definition

A point statement is used to define a point in 3D space which can then be referenced by its id.

point id ( point_triple ) endpoint

The point_triple defines the 3D position of the point. The values of each of the three coordinate must be separated by spaces.

Example

point p1 (0 0 1) endpoint

Defines a point named p1 which lies 1 unit from the origin along the z-axis..

Face

NOME Definition

A face statement is used to define a planar convex polygon which can then be referenced by its id. A face is defined by a list of points that enumerates the vertices of the face in counterclockwise or clockwise order around the face contour.

face id ( point_idlist ) [surface surface_id] endface

The point_idlist is a list of points id. The values of each of the points id must be separated by spaces. The surface_id is an optional parameter that is needed only if the face must inherit the properties of a surface.

Example

face triFace (p1 p2 p3) surface sBlue endface

Defines a face named triangleFace with vertices p1, p2, and p3, inheriting the properties of the surface sBlue.

Polyline

NOME Definition

A polyline statement generates 3D piecewise linear segments.

polyline id (point_idlist) [closed] endpolyline

The point_idlist is a list of control points. The list of points must be of length at least 2.

Example

polyline rim (v1 v2 v3 v4 v5 v6 v7 v8) endpolyline

Defines a polyline with id rim and control points v1, v2, v3, v4, v5, v6, v7, and v8..
